# Vue 3 学习笔记 - v2

## 核心概念与重点

### 1. 响应式系统

#### ref vs reactive
- **ref**：
  - 用于基本类型和对象
  - 访问/修改值需要 `.value`
  - 可以直接重新赋值整个对象
  - **推荐**：用于基本类型和需要重新赋值的对象

- **reactive**：
  - 只用于对象
  - 直接访问属性，不需要 `.value`
  - 不能直接重新赋值整个对象（会失去响应性）
  - **推荐**：用于不需要重新赋值的复杂对象

**易错点**：修改 `reactive` 对象时，不要直接赋值新对象，应修改其属性。

#### 计算属性
- **特性**：缓存结果，依赖变化时重新计算
- **getter/setter**：支持读写操作
- **最佳实践**：用于复杂计算和派生状态

**易错点**：在计算属性中不要修改响应式数据，这会导致无限循环。

### 2. 模板语法

#### 动态参数
- 使用 `:[attributeName]` 绑定动态属性
- 使用 `@[eventName]` 绑定动态事件

#### v-if vs v-for
- **优先级**：v-if 优先级高于 v-for
- **问题**：v-if 无法访问 v-for 的变量
- **解决方案**：使用 template 包装 v-for

#### 事件修饰符
- **顺序重要**：
  - `@click.prevent.self`：阻止元素及其子元素的默认行为
  - `@click.self.prevent`：只阻止元素本身的默认行为

- **常用修饰符**：
  - `.stop`：阻止冒泡
  - `.prevent`：阻止默认行为
  - `.once`：只执行一次
  - `.self`：只触发元素本身的事件

### 3. 表单处理

#### v-model
- **双向绑定**：自动同步表单值和数据
- **修饰符**：
  - `.lazy`：失去焦点时更新
  - `.number`：自动转换为数字
  - `.trim`：去除首尾空格

**易错点**：`type="number"` 输入框的 value 是字符串，需要使用 `.number` 修饰符。

### 4. 侦听器

#### watch vs watchEffect
- **watch**：
  - 懒执行，只在依赖变化时执行
  - 可以访问新旧值
  - 需要指定侦听对象

- **watchEffect**：
  - 立即执行，自动追踪依赖
  - 无法访问旧值
  - 适合处理副作用

**易错点**：`watchEffect` 只追踪第一个 await 之前的依赖。

#### 深层侦听器
- **注意**：`newValue` 和 `oldValue` 指向同一个对象
- **原因**：对象引用未改变，只是内部属性变化

### 5. 组件通信

#### Props
- 子组件通过 `defineProps` 定义
- 父组件通过 attribute 传递
- **类型验证**：建议为 props 添加类型定义

#### 事件
- 子组件通过 `defineEmits` 定义或直接 `$emit`
- 父组件通过 `@事件名` 监听
- **事件名**：建议使用 kebab-case

#### 插槽
- **作用**：父组件向子组件传递内容
- **类型**：
  - 默认插槽
  - 命名插槽
  - 作用域插槽（子组件向父组件传递数据）

**易错点**：插槽内容的作用域是父组件，不是子组件。

### 6. 模板引用

#### 基本用法
- 使用 `ref()` 创建模板引用
- 在 `onMounted` 中访问 DOM 元素
- **时机**：只能在组件挂载后访问

#### Vue 3.5+
- 使用 `useTemplateRef()` 获得更好的类型支持

### 7. 数组操作

#### 响应式数组方法
- 触发更新：`push`、`pop`、`shift`、`unshift`、`splice`、`sort`、`reverse`
- 不触发更新：`filter`、`map`、`reduce`、`concat`、`slice`（返回新数组）

**易错点**：在计算属性中使用 `reverse()` 和 `sort()` 时，要先创建数组副本。

## 最佳实践

### 1. 组件设计
- **单一职责**：每个组件只负责一个功能
- **Props 向下，Events 向上**：数据通过 props 传递，行为通过 events 通知
- **插槽定制**：使用插槽实现内容的灵活定制

### 2. 性能优化
- **v-if vs v-show**：频繁切换用 v-show，条件稳定用 v-if
- **计算属性缓存**：复杂计算使用 computed
- **避免模板中的方法调用**：方法会在每次渲染时执行
- **合理使用侦听器**：避免不必要的 watch

### 3. 代码风格
- **组件命名**：PascalCase（文件名和组件名）
- **模板引用**：kebab-case（模板中的 ref 属性）
- **事件名**：kebab-case
- **代码组织**：逻辑放在 script 中，模板保持简洁

### 4. 类型安全
- **TypeScript**：使用 TypeScript 增强类型安全性
- **Props 类型**：为 props 添加类型定义
- **Emits 类型**：为事件添加类型定义

## 常见问题与解决方案

### 1. 响应式数据问题
- **问题**：修改对象属性后视图不更新
- **原因**：可能是对象不是响应式的，或直接赋值了新对象
- **解决方案**：使用 `reactive` 或 `ref`，修改属性而不是赋值新对象

### 2. 组件通信问题
- **问题**：子组件无法向父组件传递数据
- **解决方案**：使用事件或作用域插槽

### 3. 模板引用问题
- **问题**：模板引用为 null
- **原因**：在组件挂载前访问了模板引用
- **解决方案**：在 `onMounted` 中访问，或使用条件判断

### 4. 计算属性问题
- **问题**：计算属性不更新
- **原因**：依赖的响应式数据没有变化，或计算属性中修改了依赖
- **解决方案**：检查依赖是否正确，确保计算属性是纯函数

### 5. 事件处理问题
- **问题**：事件修饰符不生效
- **原因**：修饰符顺序不正确
- **解决方案**：注意修饰符的顺序，如 `.stop.prevent`

## 高级特性

### 1. 组合式 API
- **优势**：逻辑复用，代码组织更灵活
- **最佳实践**：使用 `useXXX` 命名规范封装可复用逻辑

### 2. 异步组件
- **用法**：`defineAsyncComponent`
- **场景**：大型组件的懒加载

### 3. 自定义指令
- **用法**：`app.directive`
- **场景**：DOM 操作的复用

### 4. 插件系统
- **用法**：`app.use`
- **场景**：全局功能的扩展
